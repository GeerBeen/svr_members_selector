Щоб отримати список кандидатів відібраних в раду треба

1. запустити сервер ```python -m backend.app``` (приклад для запуску з кореневої директорії)
2. виконати POST-запит на /get-council ```fetch("http://localhost:5000/get-council", ...)``` передавши в нього джейсон з наступними полями:
    - amount - кількість кандидатів, яку треба вибрати
    - specialty_id - id спеціальності заявника
    - keywords - список ключових слів
    - key (опціонально) - тип підбору ([нижче](#key) детальніше розписано що це таке)

    Він має виглядати приблизно так (аргументи обрані для прикладу):
```
{
    "amount": 5,
    "key": 0,
    "specialty_id": "01.01.01",
    "keywords": ["math", "algebra"]
}   
``` 


Запит повертає json з подібним змістом:
```
{
    "orcid_list": [
        "0000-0002-1665-0361",
        "0000-0001-9342-8863",
        "0000-0003-0849-057X",
        "0000-0001-8330-2909",
        "0000-0003-1730-7140"
    ]
}
```
Це orcid'шники кандидатів вибраних в раду

<a id="key"></a>
## key
key - тип кандидатів. Алгоритм вибере в раду, тільки тих кандидатів, що відповідають заданому типу.

Значення:
- key=0 - __кандидати зі спеціальності ідентичної__ спеціальності заявника
- key=1 - __кандидати з ідентичної або суміжної спеціальностей__ по відношенню до спеціальності заявника
- key=2 (або будь яке інше значення) - __кандидати з усіх можливих спеціальностей__

__Примітка:__ для значення key=1 функціонал не готовий (і скоріш за все не буде). на даному етапі там стоїть заглушка і при значенні key=1, програма поводить себе аналогічно до значення key=0

Також можна не передавати key, тоді його значення автоматично встановиться на key=0
